# D4 Reference — PI Staging Instructions

`papers/diagnostic/d4_reference.wav` and `papers/diagnostic/d4_reference.lyrics.txt`
are required for M0 D4 (segmentation + Demucs vocal isolation + Whisper WER smoke).
This audio is **not** generated by the system — it must be a real human-vocal track
that the PI provides. Claude must not fabricate it.

## What to provide

1. **`papers/diagnostic/d4_reference.wav`**
   - A short vocal track with clearly intelligible vocals
   - **Duration**: 30–90 seconds (≥30 s required; longer is fine but inflates D4 wall time)
   - **Sample rate**: 44.1 kHz or 48 kHz (`save_audio` / `load_audio` handle either)
   - **Channels**: mono or stereo, both OK
   - **Format**: PCM-encoded `.wav` (uncompressed). MP3/FLAC will fail `torchaudio.load`
     unless backend is configured; stick to `.wav`.
   - **Content**: vocals + minimal accompaniment is fine, but Demucs needs to be able
     to isolate the vocal stem cleanly. Acapella or vocals-with-light-accompaniment
     is best.

2. **`papers/diagnostic/d4_reference.lyrics.txt`**
   - Plain UTF-8 text
   - Must match the audio (Whisper-WER compares ASR transcript against this text)
   - One verse / line per line; no JSON, no markdown, no front-matter
   - Same language as the vocals; English preferred for the first audit pass

## Copyright + privacy

- **Do not stage copyrighted commercial recordings.** Use one of:
  - Original recording (PI-authored or with consent), OR
  - Public-domain / CC0 vocal sample (e.g. LibriVox, Free Music Archive CC0 vocals,
    or a Common Voice fragment), OR
  - Self-recorded a-cappella vocals (30 s of humming + a verse is sufficient).
- **Do not stage anything that identifies a non-consenting third party.**

## Acceptance criteria for the audit

After staging both files, run:

```bash
conda activate audio-prm
PYTHONPATH=src python -c "
from mprm.data.audio_io import load_audio
w, sr = load_audio('papers/diagnostic/d4_reference.wav')
import math
rms_db = 20 * math.log10(max(1e-12, float(w.float().abs().mean())))
print(f'  shape={tuple(w.shape)} sr={sr} dur={w.shape[-1]/sr:.1f}s rms_db={rms_db:.1f}')
"
```

Expected output: non-NaN, duration in [30, 120] s, `rms_db > -40` (non-silent).

## How M0 uses these files

`scripts/launch_phase_a.sh` (M0 D4 block) calls:

```bash
python scripts/d4_segmentation.py --audio papers/diagnostic/d4_reference.wav \
    --lyrics "$(cat papers/diagnostic/d4_reference.lyrics.txt)"
```

which exercises Demucs vocal isolation, the section-boundary heuristic, and a
Whisper WER smoke against the staged lyrics. M0 D4 production blocks with exit
2 if either file is missing.

## Staging step

```bash
cp <your-recording>.wav        papers/diagnostic/d4_reference.wav
cp <your-lyrics>.txt           papers/diagnostic/d4_reference.lyrics.txt
rm papers/diagnostic/MISSING_D4_REFERENCE.flag
```

Do **not** remove `MISSING_D4_REFERENCE.flag` before both files are staged and
the acceptance check above passes.

## A note on DEV-ONLY synthetic placeholders

A synthetic placeholder is NOT staged by this preparation pass. Synthetic audio
(e.g. ACE-Step-generated samples) would defeat the purpose of D4 (the audit's
job is precisely to verify Demucs + Whisper on **real** vocals). If you want a
quick MODE=dev pass to test the script plumbing, `scripts/launch_phase_a.sh`
already falls back to the D2-generated instrumental sample without WER check —
no separate DEV-ONLY reference file is required for that path.
